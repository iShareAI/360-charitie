Our Process · workflow

Stage One — Intake & Analysis

Goal: Gather all source materials and understand the project well enough to convert them accurately into 3D.

What we collect

  • Architectural drawings: floor plans, sections, elevations, details, MEP/structural (if available).
  • References: site photos, finishes schedules, mood boards, material specs (R-values, colors, textures), furniture lists.
  • CAD/BIM files: DWG/DXF, RVT/IFC, PDFs (to scale), plus any coordinate system notes.
  • Constraints: target LOD (e.g., LOD 200/300), performance budget (polycount), target platform (UE5, WebGL, mobile), deadlines.

What we do

  • Verification & scaling — Check dimensions, drawing scale, and units; confirm north, levels, and key datums.
  • Clash/consistency check — Cross-check plans vs. elevations/sections; note missing heights, window schedules, stair details, etc.
  • Scope definition — Define inclusion list (shell, interior partitions, FF&E, landscape), LOD per category, render vs. real-time needs.
  • Delivery plan — Choose pipeline (e.g., Revit → IFC → Blender → Unreal), file formats, naming conventions, and review milestones.

Deliverables

  • Intake report with questions/RFIs.
  • Confirmed scope + LOD matrix.
  • Schedule with milestones and review checkpoints.

Stage Two — Conversion from 2D to 3D

Goal: Build an accurate, optimized 3D model from 2D drawings suitable for visualization, coordination, or real-time use.

Inputs

  • Clean, scaled 2D drawings (DWG/DXF/PDF), reference photos, material schedules.

Pipeline (high level)

  • Set up the project — Establish units, origin, levels, and grids. Import DWGs on locked reference layers.
  • Massing & structural shell — Create slabs, walls, roofs from plan/section profiles; apply true heights from levels.
  • Openings & components — Add doors, windows, stairs, railings from schedules/details; keep parametric sizes when possible.
  • Interior build-out — Partitions, ceilings, built-ins, sanitary ware, casework. Place key FF&E for scale/context.
  • Site & context (if in scope) — Terrain from contours, adjacent masses, curb lines, vegetation placeholders.
  • Materials & UVs — Logical material names (e.g., Floor_Tile_600x600), UV scale matches real size.
  • Optimization for target platform — Instance repeats, generate LODs, lightmap UVs for real-time, tidy naming (Category_Type_Size_V##).
  • QA & dimensional checks — Spot‑measure spans, door heights, stair risers; validate vs. drawings/RFIs.
  • Packaging & export — Deliver native file + exports (FBX/GLB/IFC/UAsset) and a readme (versions, units).

Standards & conventions

  • Units: meters (or client spec); origin at project base point.
  • Layers/Collections: Architecture / Structure / MEP / FF&E / Site.
  • Naming: A_Wall_Int_100mm_V01, A_Window_1200x1500_V02.

LOD targets (example)

  • Exterior shell: LOD 300 (accurate dims, simplified trims).
  • Interior: LOD 200–300 depending on review needs.
  • FF&E: hero items LOD 300; background LOD 150–200.

Quality checks before delivery

  • No non‑manifold geometry; normals unified.
  • Pivot at logical placement point; transforms frozen.
  • Real‑world scale verified (e.g., door 2.10 m).
  • Material count reasonable; texel density consistent.
  • File opens cleanly without missing links.

Deliverables

  • 3D model (native + agreed export).
  • Material list (CSV/PDF) mapping to the model.
  • QA report (assumptions, open RFIs).
  • Optional: preview renders or lightweight viewer (GLB).

Stage Three — Rendering (Lookdev, Lighting & Output)

Goal: Produce photoreal (or stylized) stills, 360° panoramas, and/or animation from the approved 3D model.

Inputs

  • Approved 3D scene with materials + UVs, scale verified.
  • Style direction (references/mood), time-of-day, weather, occupancy level.
  • Brand/CI constraints (color space, framing, overlays, logo).

Pipeline Overview — Track A (Real‑time · Unreal Engine 5)

  • Scene prep: Nanite where suitable; Lumen GI/Reflections; Virtual Shadow Maps; ACES tonemapping.
  • Lighting: Sun & Sky, HDRI for overcast/night; Rect Lights for interiors.
  • Materials: PBR maps (BaseColor/Roughness/Metallic/Normal/AO); adjust clearcoat/subsurface; verify texel density.
  • Cameras: Sequencer; filmback (36×24 mm), lenses 18–85 mm; physical exposure; autofocus/rack focus as needed.
  • Quality: Screen percentage 150–200% for stills; MRQ AA (TAAU/TSR + Temporal Samples 8–16).
  • Output: Stills EXR 16‑bit; animations ProRes 422 HQ or EXR/PNG; panoramas 8–12K.

Pipeline Overview — Track B (Offline · V‑Ray / Corona / Cycles)

  • Lighting: Physical Sun/Sky or calibrated HDRIs; interior area lights; portals (if engine uses them).
  • Sampling: 256–1024 spp; denoise (OIDN/OptiX 0.3–0.5); stable seed & light clamping for anim.
  • AOVs/Passes: Beauty, Diffuse, Reflection, Refraction, Emission, SSS, Specular, Cryptomatte, Z‑Depth, Position, Normal.
  • Output: EXR 16‑bit/32‑bit (ACEScg); graded PNG/TIFF & ProRes/H.264 deliveries.

Look Development

  • WB & contrast via ACES; avoid clipped highlights.
  • Material tuning: IORs, roughness ranges; scale‑corrected textures; clean tiling/seams.
  • Set dressing (scope‑dependent): plants, people, props; hero assets at higher LOD.

Composition & Motion

  • Eye‑level 1.50–1.65 m for interiors; orthographic elev/section as needed.
  • Rule of thirds, leading lines; avoid extreme distortion unless intentional.
  • Animation: 5–12 s shots; ease in/out; 0.3–0.6 m/s dolly; gentle parallax; 180° rule.

QA Checklist (pre‑delivery)

  • No light leaks/fireflies; consistent exposure.
  • Correct shadow softness & contact shadows.
  • Clean normals; no flipped faces or floats.
  • Consistent texel density; no stretching.
  • Color pipeline verified (ACES → sRGB/Rec.709).

Post‑Production

  • Grade in ACES (lift/gamma/gain); subtle bloom/glare; vignette when needed.
  • Depth of field, motion blur (optical or in‑engine).
  • Compositing from AOVs (Cryptomatte masks); sky replacements if specified; overlays/branding.

Deliverables

  • Stills: 4K PNG/TIFF + web JPEGs; optional 8K hero views.
  • 360°: 8–12K EXR/JPEG equirectangular; optional WebVR viewer.
  • Animation: 1080p/4K master (ProRes 422 HQ), web H.264/H.265 15–40 Mbps.
  • Source: EXR sequences, LUTs, project files (on request), shot list & contact sheet.

Reviews & Milestones (example)

  • Lookdev frames (6–10 stills) — lighting/materials approval.
  • Anim previz (playblasts / low‑sample) — camera paths & timing.
  • Draft render — 80% quality for notes.
  • Final render & grade — delivery + one revision pass for color/levels.

Stage Four — Cinematic Walkthrough (film through the interior)

At this stage, we create a smooth, story‑driven walkthrough that highlights circulation, light, and materials while conveying the design intent.

Pre‑production (planning)

  • Route & structure: entry → key moments → final frame (60–120 s per level).
  • Shot list: establishing → reveals → hero angles → detail inserts → closing. Mark door opens/light changes.
  • Tempo & music: pick BPM to match camera speed; cut on bars.
  • Time of day: lock one TOD for continuity (e.g., golden hour).
  • Tech constraints: target quality (4K/30 or 4K/60), poly/texture budget; enable level streaming if needed.

Production (Unreal Engine 5)

Project & cameras
  • Master Sequence + sub‑sequences by zone.
  • CineCameraActor, filmback 36×24 mm, lenses 18–85 mm.
  • ACES on; manual exposure (no auto).
Camera motion
  • Camera on Spline/Cine Rail (dolly).
  • Speeds: interiors 0.3–0.6 m/s; corridors up to 0.8 m/s.
  • Turns ≤ 45°/s; no shake unless intentional.
  • Focus tracking on targets; rack focus for emphasis only.
Lighting & look
  • Lumen GI/Reflections, Virtual Shadow Maps.
  • Directional + Sky Atmosphere; HDRI for night/overcast; Rect/IES for interiors.
  • Consistent color temps (e.g., 3000 K warm, 5600 K daylight).
  • Materials: correct roughness/normal scale so highlights read cleanly.

Micro‑animation (optional)

  • Doors/curtains/water/screens; subtle wind on plants.
  • Hero assets high quality; background instanced.

Rendering

  • Movie Render Queue: 4K (3840×2160), 30 fps (or 60 fps for fast motion).
  • AA High, 8–16 temporal samples; Screen Percentage 150–200%.
  • Motion blur on (~180° shutter). Format: EXR 16‑bit (ACEScg) or ProRes 422 HQ for previews.

Post‑production

  • Edit rhythm: cut on action/music; alternate wide → medium → detail.
  • Audio: ambience + light foley + music; deliver around −16 LUFS (web).
  • Grading: ACES → Rec.709; gentle contrast, protect highlights.
  • Graphics: intro/outro, logo, lower‑thirds; optional plan overlay.

QA checklist

  • No clipping through geometry; door swings clear.
  • Stable exposure; no auto‑exposure jumps.
  • No light leaks/fireflies; consistent shadows.
  • Clean normals/UVs; even texel density.
  • Smooth path; no jerks or sudden speed changes.
  • Color pipeline verified (ACES → sRGB/Rec.709).

Stage Five — Naming Opportunities (Objects & Placements)

At this stage, we create a mapped, tiered system of physical & digital placements for sponsor/donor names—tested in 3D for scale, visibility, and accessibility.

At this stage, we

  • Audit the environment and map high‑impact, high‑visibility placements for sponsor/donor names.
  • Define sponsorship tiers and build a size/material/lighting tier matrix aligned to fundraising goals.
  • Design tangible & digital objects (plaques, donor walls, feature walls, signage, floor inlays, digital displays, sculptural markers).
  • Prototype objects in 3D inside the approved scene to test scale, sightlines, and legibility at real eye heights (~1.5–1.65 m).
  • Validate accessibility & code (mounting heights, contrast, clearances, fixings, edge radii).
  • Establish brand rules (typography sizes, clear‑space, contrast ratios, max name length, multilingual handling).
  • Build content templates (parametric text; CSV‑driven variants) for rapid name updates.
  • Produce specs & shop drawings; deliver finalized assets and an update guide.

We use (inputs)

  • Approved 3D model & plans (zones, circulation, key sightlines).
  • Brand kit (logo, colors, typography, tone).
  • Sponsorship tiers + legal/contract notes.
  • Accessibility & code requirements (ADA/ISO, fire safety, mounting heights).

We produce (deliverables)

  • Annotated location plans with proposed placements (entries, lobbies, corridors, feature walls, rooms, outdoor nodes).
  • Object designs and 3D previews (in‑context renders + short UE5 fly‑throughs).
  • Tier matrix with sizes, materials, illumination, and price bands (Platinum/Gold/Silver/Room/Seat).
  • Specs & shop drawings (PDF/DWG) and a brand usage guide.
  • Assets & templates: instanced UE5 objects with updatable names; CSV/JSON templates.

How we do it (process)

  • Opportunity audit → sightlines, dwell points, hero walls; tag by impact/traffic.
  • Tiering & matrix → object types, dimensions, finishes, lighting, budgets per tier.
  • Concept design → wall‑mounted / freestanding / integrated; materials & lighting choices.
  • Prototype in 3D → place in scene; run visibility/legibility checks from approach angles.
  • Accessibility & compliance → heights, contrast/tactile needs, egress clearances, fixings.
  • Documentation → annotated sheets, exploded views, BOMs, finish schedules.
  • Handover → export assets, provide licenses/substitutions, update guide.

Example tier matrix (excerpt)

  • Feature Wall (Platinum): 3000×1800 mm panel; brushed metal + backlit glass letters; dimmable LED.
  • Room Naming (Gold): 600×300 mm plaque; anodized aluminum + engraved infill; non‑glare finish.
  • Zone Marker (Silver): 400×200 mm plaque; solid surface + UV print; standoffs.
  • Seat/Bench (Bronze): 120×40 mm plate; stainless steel, laser‑etched; tamper‑resistant screws.
  • Digital Spot: 55″ portrait display; donor rotation; CMS‑ready.

Technical guidelines we follow

  • Typography: brand font; x‑height for 5 m read ≈ 60–80 mm; sensible fallbacks.
  • Contrast: WCAG AA ≥ 4.5:1; glare‑controlled finishes.
  • Lighting: 200–400 lx on plaque; avoid specular hotspots; night‑mode curves.
  • Durability: exterior IP ratings; vandal‑resistant fasteners; replaceable plates.
  • Data workflow: CSV/JSON → auto‑layout templates → versioned exports.

Stage Six — Data Population (filling the model with data)

At this stage, we populate the 3D/BIM model with structured metadata so it’s searchable, exportable, and ready for operations, fundraising, and visualization pipelines.

Inputs

  • Approved 3D model (BIM/DCC/UE5 scene).
  • Source datasets: room/area schedules, FF&E lists, materials, donor registry, signage matrix, maintenance & cost tables, localization strings.
  • Standards to follow (e.g., IFC, COBie, ISO 19650 naming, Uniclass/OmniClass).

What we add (key data domains)

  • Spaces & rooms: IDs, names, functions, floor/zone, areas (GIA/NIA), accessibility data.
  • Architectural elements: type/instance codes, materials, fire ratings, acoustic values, U/R‑values, manufacturers.
  • FF&E & equipment: model numbers, warranties, maintenance intervals, asset barcodes/QR links.
  • Signage & naming opportunities: tier, donor name, language, character limits, approval status.
  • Cost & phase: WBS/CSI codes, procurement status, install phase, replacement cost.
  • Wayfinding & geodata: coordinates, path IDs, nearest exits, visibility tags for tours.
  • Localization: multilingual labels and strings (EN/ES/…).

Data schema & governance

  • Schema: IFC Psets + custom properties, COBie fields where required.
  • Unique IDs: stable GUID/UID per element; no duplicates; versioned change log.
  • Naming convention: ISO 19650‑style file/asset names (e.g., A_Wall_Int_100mm_V03).
  • Validation rules: required fields, allowed values, ranges, regex patterns.

Mapping & automation

Field mapping
  • Create a mapping table (Source → Target):
  • CSV.RoomName → IfcSpace.Name
  • CSV.DonorName → Signage_BP.DonorText
  • CSV.MaintenanceInterval → Asset.MaintenanceDays
Batch population
  • BIM (Revit/IFC): Shared Parameters, Type/Instance params, Classification Manager; export IFC with mapped Psets.
  • DCC (Blender): Custom properties on Collections/Objects via Python; inherit to instanced assets.
  • Unreal Engine 5: DataTables (CSV/JSON), Primary Data Assets, actor Tags/Metadata; Blueprint components to read rows at begin play.
Automation tools
  • CSV/JSON importers, pyRevit/Revit API or Blender Python scripts, UE5 Editor Utility Widgets for bulk assign.
  • Optional live link to Google Sheets/CSV (pull → validate → write).

Quality control (before sign‑off)

  • Completeness: no empty required fields; all rooms/elements have IDs.
  • Consistency: units in meters; value ranges (e.g., door height ≈ 2.10 m); enums match schema.
  • Uniqueness: no duplicate room numbers, no reused GUIDs.
  • Cross‑checks: donor names meet character limits; multilingual text renders correctly (accents, RTL if any).
  • Exports test: IFC opens with properties intact; COBie sheet passes validator; UE5 reads DataTables without missing keys.

Integration with visualization & ops

  • UE5 hooks: Blueprint labels/tooltips/filters (e.g., filter "Gold tier").
  • Cinematics auto‑pull donor names and room titles per shot list.
  • Interactive tours filter spaces by function, floor, accessibility.
  • QR codes (optional): per‑asset URL (asset/{UID}); scan → maintenance/donor page.
  • Analytics (optional): tag hotspots for heatmaps or user flows in demos.
